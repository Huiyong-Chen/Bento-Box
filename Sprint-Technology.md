# 🎯 Sprint 技术学习地图 (Technology Learning Map)

**目标**：分阶段、有针对性地掌握架构师所需技术栈，避免学习疲劳和信息过载

**原则**：每个阶段只学习"刚好够用"的技术深度，重点理解架构思维而非技术细节

---

## 📊 学习深度定义

| 深度等级 | 描述 | 学习目标 | 时间分配 |
|---------|-----|---------|---------|
| **了解** | 知道是什么，能基本使用 | 概念理解 + 简单实践 | 1-2天 |
| **熟悉** | 能独立使用，理解核心原理 | 动手实践 + 问题解决 | 3-5天 |
| **精通** | 能优化和扩展，理解内部机制 | 源码阅读 + 高级应用 | 1-2周 |

## 🟢 Sprint 1：架构骨架与规范统一

### 🎯 阶段目标
建立 Monorepo 物理结构，掌握配置复用模式

### 📚 核心技术栈

#### 1. **pnpm Workspace** ⚡
- **学习深度**: **熟悉**
- **为什么**: Monorepo 的核心基础设施，决定了整个项目的依赖管理策略
- **学习重点**:
  - workspace 协议语法 (`workspace:*`)
  - 依赖提升机制 (`shamefully-hoist`)
  - 软链 vs 硬链的区别
  - workspace 配置文件语法
- **掌握标准**: 能配置复杂的 workspace 依赖关系
- **时间**: 2天

#### 2. **TypeScript 配置** 🔧
- **学习深度**: **熟悉**
- **为什么**: TS 是现代前端的基石，Project References 是 Monorepo 的关键
- **学习重点**:
  - `composite: true` 的作用
  - Project References 语法
  - 路径映射 (`paths`)
  - 增量编译原理
- **掌握标准**: 能配置多包间的 TS 引用关系
- **时间**: 3天

#### 3. **ESLint 配置** 📏
- **学习深度**: **了解**
- **为什么**: 需要统一代码规范，但不必深入自定义规则
- **学习重点**:
  - Flat Config 语法 (ESLint 9)
  - 规则继承和扩展
  - 插件系统基础
  - 自动修复机制
- **掌握标准**: 能配置基本的代码检查规则
- **时间**: 1天

#### 4. **Monorepo 概念** 🏗️
- **学习深度**: **熟悉**
- **为什么**: 架构思维的核心，理解单仓多包的设计哲学
- **学习重点**:
  - Monorepo vs Multi-repo 的优劣
  - 依赖拓扑管理
  - 版本管理和发布策略
  - 代码共享模式
- **掌握标准**: 能解释 Monorepo 的价值和适用场景
- **时间**: 2天

### 📅 时间分配：**8天**
### 🎯 验收标准：能独立搭建基础的 Monorepo 项目结构

## 🟡 Sprint 2：核心基建与 SSO 模拟

### 🎯 阶段目标
实现"登录一次，到处运行"的 SDK 封装

### 📚 核心技术栈

#### 1. **Axios (HTTP 客户端)** 🌐
- **学习深度**: **熟悉**
- **为什么**: 网络请求是前后端交互的核心，需要封装统一的请求层
- **学习重点**:
  - 请求/响应拦截器
  - 错误处理和重试机制
  - 请求取消 (AbortController)
  - 基础配置 (超时、headers)
- **掌握标准**: 能封装企业级的 HTTP 客户端
- **时间**: 3天

#### 2. **JWT & 认证概念** 🔐
- **学习深度**: **了解**
- **为什么**: SSO 的基础知识，理解认证流程设计
- **学习重点**:
  - JWT 的结构 (Header.Payload.Signature)
  - Token 存储策略 (LocalStorage vs Cookie)
  - 刷新令牌 (Refresh Token) 机制
  - 过期处理和自动刷新
- **掌握标准**: 理解 SSO 的基本工作原理
- **时间**: 2天

#### 3. **Mock Service Worker (MSW)** 🎭
- **学习深度**: **了解**
- **为什么**: 前后端分离开发时的 API 模拟工具
- **学习重点**:
  - 请求拦截和响应 Mock
  - RESTful API 模拟
  - 动态响应和延迟模拟
  - 开发环境集成
- **掌握标准**: 能为开发环境搭建基本的 API Mock
- **时间**: 2天

#### 4. **前端状态管理基础** 📦
- **学习深度**: **了解**
- **为什么**: 需要管理用户认证状态
- **学习重点**:
  - 组件状态 vs 全局状态
  - Context API 或简单 store
  - 状态持久化
  - 状态同步机制
- **掌握标准**: 理解状态管理的必要性和基本模式
- **时间**: 2天

### 📅 时间分配：**9天**
### 🎯 验收标准：完整的登录流程能正常工作，Token 管理正确

## 📊 总体学习策略

### 🎯 学习原则
1. **深度优先于广度**: 每个技术只学"刚好够用"的深度
2. **实践驱动学习**: 先用起来，再深入理解原理
3. **架构思维先行**: 重点理解"为什么这么设计"
4. **渐进式掌握**: 从了解→熟悉→精通逐步深入

### 📅 时间规划
- **总时间**: 3-4个月 (61天学习时间)
- **节奏**: 每周3-4天技术学习，1-2天实践项目
- **调整**: 根据个人进度灵活调整每个阶段的时间

### 📚 学习资源推荐
- **官方文档**: 优先学习官方文档，建立正确认知
- **实践项目**: 边学边做，理论联系实际
- **源码阅读**: 理解内部实现机制
- **社区讨论**: 关注技术选型的权衡和取舍

### 🎉 成功标志
- **能独立设计**: 中小型项目的完整架构
- **能合理取舍**: 在技术选型中做出明智决策
- **能持续学习**: 建立终身学习的习惯和能力
- **能团队协作**: 与后端、前端、运维有效配合

**记住**: 架构师的成长是一个持续的过程，重点是建立正确的思维框架和决策能力！🚀
